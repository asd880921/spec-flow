# codebase-design-doc-cleaner

當使用者要求清洗、定稿、交件或準備釋出由 `codebase-sa-builder`（架構層級 SA 文件）或 `codebase-sa-sd-finalizer`（SA + SD 混合文件 `*-sd.md`）產出的設計文件時，使用此 skill。

本 skill 只移除分析過程中有用、但對讀者造成干擾的內部工作註記。**直接編輯該 Markdown 文件本身**。除非使用者明確要求自動化，否則不要建立或執行清洗腳本。不得改寫需求、不得重新設計技術內容、不得補上缺漏的分析，也不得修改任何其他 skill。

## 操作對象與安全邊界

- 清洗對象是**交付用的設計文件**：`codebase-sa-sd-finalizer` 產出的 `*-sd.md` 混合文件、`codebase-sa-builder` 產出的 SA 文件，或拆分情境下的**母文件（拆分總覽 SA）**。
- **一次清一份，由使用者個別調用。** 拆分情境（1 母 N 子）下，使用者會對母文件與每份子文件逐一執行本 skill；本 skill 不主動批次處理整組文件，只清使用者本次指定的那一份。
- **不要**動到 `codebase-sa-sd-finalizer` 流程中保持原狀的原始 SA 文件（即未加 `-sd` 的原檔）。清洗只針對使用者指定要交付的那一份。
- 若使用者希望保留清洗前的版本，先確認，再依其指示另存乾淨版或就地編輯。

## 清洗範圍

只移除以下項目：

- 整個 `## 來源標記說明` 章節。
- 整個 `## 待釐清事項` 章節（新版混合文件與 SA 文件使用此名稱）。
  - 相容處理：若文件仍使用舊章節名 `## 待釐清問題` 或 `## 設計待釐清問題`，一併移除。
- 文件內文中的來源標記：
  - 名為 `來源` 的 Markdown 表格欄位（整欄移除，保留其他欄位與所有列）。
  - 行首前綴，例如 `PRD-confirmed：`、`SA-confirmed：`、`Code-confirmed：`、`Doc-supplement：`、`Inferred：`、`Design-decision：`、`Unknown：`（含相容的舊標記 `Issue-confirmed：`）。
  - 獨立成行的來源標記，例如 `來源：SA-confirmed / Code-confirmed。`。

## 必須保留的內容

保留所有面向讀者的內容，包括：

- `閱讀導覽`。
- 前半段收斂後的 SA 架構脈絡：背景與目標、範圍、角色與權限、功能需求、業務規則、流程、架構概觀、資料與整合概念等。
- 後半段的 SD 系統設計內容：設計摘要、核心設計決策、資料設計與 TableSchema、額外索引／後續評估索引、系統流程與資料流、模組互動與介面契約、權限與稽核設計、實作注意事項、測試重點等。
- 最後一章整合後的**驗收條件**（混合文件的總驗收條件）。其表格中的 `層面`（SA / SD）等欄位屬於讀者內容，**予以保留**；只移除 `來源` 欄位。
- **母文件（拆分總覽 SA）的核心章節**：`模組／功能分解`、`跨模組業務規則與共用概念`、`跨模組／端對端驗收條件`，以及指回各模組的對應檔名——這些都是讀者內容，予以保留；同樣只移除其表格的 `來源` 欄位，不動其餘欄位。
- 既有的用字遣詞與章節順序，除了被移除的章節之外。
- 輔助理解的 Mermaid 圖、表格與程式識別符。

## 工作流程

1. 確認要清洗的 Markdown 檔案路徑或使用者貼上的文件內容。
2. 以一般檔案編輯工具直接編輯文件。
3. 移除目標章節：刪除每個符合的 `##` 標題，以及其後直到下一個 `##` 標題或檔案結尾的所有內容。
4. 從 Markdown 表格中移除 `來源` 欄位，保留其餘欄位與所有列（包含驗收條件表的 `層面` 欄位）。
5. 從內文移除來源標記前綴與獨立成行的來源標記。
6. 除非使用者明確要求額外編輯，否則不更動其他任何內容。
7. 驗證清洗後的文件不再包含：
   - `來源標記說明`
   - `待釐清事項`（及舊名 `待釐清問題`、`設計待釐清問題`）
   - `PRD-confirmed`、`SA-confirmed`、`Code-confirmed`、`Doc-supplement`、`Inferred`、`Design-decision`、`Unknown`、`Issue-confirmed`
   - 殘留的空白 `來源` 表格欄位或對齊分隔列
8. 抽查面向讀者的關鍵章節仍存在，尤其是最後一章的驗收條件，以及（若適用）SD 的 TableSchema 與索引章節。

## 輸出規則

- 若使用者提供檔案路徑，直接就地編輯該檔案，除非使用者要求另存一份乾淨輸出檔。
- 若使用者貼上文件內容，僅回傳清洗後的 Markdown 內容。
